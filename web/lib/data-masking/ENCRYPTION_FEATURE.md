# 映射文件加密功能

## 概述

为了保护导出的映射文件中的原始敏感数据，系统提供了基于AES-256-GCM的加密功能。用户可以使用32位加密口令对映射文件进行加密，确保即使文件泄露也无法直接读取原始敏感信息。

## 功能特性

### 1. 加密设置（默认启用）

- 位置：设置 → 数据安全 → 映射文件加密
- 默认状态：开启
- 可通过开关控制是否启用加密

### 2. 加密口令生成

支持两种方式：

**系统生成（推荐）**
- 自动生成32位十六进制随机字符串
- 高安全性，不易被破解
- 口令默认隐藏，可点击显示/复制

**用户自定义**
- 允许用户输入自定义口令
- 最少32位字符
- 实时显示口令长度

### 3. 加密流程

1. 用户完成文件脱敏并预览
2. 点击"执行脱敏并保存"
3. 如果启用了加密，显示加密设置界面
4. 选择生成或输入口令
5. 确认后，映射文件使用AES-256-GCM加密
6. 保存加密后的映射文件

### 4. 加密文件格式

```json
{
  "version": "1.0",
  "encrypted": true,
  "data": "<base64-encoded-encrypted-data>"
}
```

### 5. 解密还原流程

1. 上传脱敏文件和加密的映射文件
2. 系统检测到 `encrypted: true` 标记
3. 提示用户输入解密口令
4. 使用口令解密映射数据
5. 继续正常的反脱敏流程

### 6. 跳过加密选项

即使启用了加密功能，用户仍可在加密设置界面选择"跳过加密"，直接保存未加密的映射文件（向后兼容）。

## 安全考虑

### 优势

1. **数据保护**：原始敏感数据在映射文件中加密存储
2. **AES-256-GCM**：使用业界标准的加密算法
3. **随机IV**：每次加密使用不同的初始化向量
4. **认证加密**：GCM模式提供完整性验证

### 注意事项

⚠️ **口令管理**
- 加密口令无法恢复
- 口令丢失后无法解密映射文件
- 建议使用密码管理器保存口令

⚠️ **向后兼容**
- 系统支持未加密的映射文件（旧版本）
- 检测到未加密文件时直接使用

## 配置管理

### 存储位置

- **LocalStorage**: `mapping_encryption_enabled`
- **数据库**: 用户配置表 `user_config`

### 默认值

- 新用户：默认启用（`true`）
- 升级用户：默认启用（`true`）

### 配置同步

- 配置保存到数据库
- 跨设备、跨浏览器同步
- 刷新页面不丢失

## 使用示例

### 启用加密导出

```typescript
// 1. 在设置中启用加密（默认已启用）
localStorage.setItem('mapping_encryption_enabled', 'true')

// 2. 完成文件脱敏
// 3. 系统自动显示加密设置界面
// 4. 生成或输入口令
// 5. 确认并保存
```

### 禁用加密导出

```typescript
// 1. 在设置中关闭加密
localStorage.setItem('mapping_encryption_enabled', 'false')

// 2. 完成文件脱敏
// 3. 直接保存未加密的映射文件
```

### 解密还原

```typescript
// 1. 上传加密的映射文件
// 2. 系统检测到加密标记
// 3. 输入解密口令
// 4. 解密成功后继续还原
```
