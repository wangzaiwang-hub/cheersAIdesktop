# 全局加密口令功能使用指南

## 问题：看不到口令输入框

如果您在"设置 → 数据安全"页面中看不到"全局加密口令"输入框，请按照以下步骤操作：

### 方法 1：刷新浏览器（推荐）

1. 在浏览器中按 `Ctrl + Shift + R`（Windows/Linux）或 `Cmd + Shift + R`（Mac）进行硬刷新
2. 或者按 `Ctrl + F5`（Windows/Linux）
3. 这会清除缓存并重新加载页面

### 方法 2：重新构建前端

如果刷新后仍然看不到，需要重新构建前端：

```bash
cd web
pnpm install
pnpm build
```

然后重启开发服务器。

## 功能位置

更新后，您应该能在以下位置看到新功能：

### 设置 → 数据安全页面

1. 点击右上角的用户头像
2. 选择"设置"
3. 在左侧菜单选择"数据安全"
4. 找到"映射文件加密"部分
5. 确保开关是**开启状态**（蓝色）
6. 开关下方会显示：
   - ✓ 提示信息
   - **全局加密口令**输入框
   - "显示/隐藏"按钮
   - "生成32位口令"按钮
   - "保存口令"按钮
   - "复制"按钮（在有口令时显示）
   - ⚠️ 警告信息

## 使用流程

### 1. 配置全局口令

1. 进入"设置 → 数据安全"
2. 确保"映射文件加密"开关是开启的
3. 点击"生成32位口令"按钮（或手动输入至少32位字符）
4. 点击"保存口令"按钮
5. 建议点击"复制"按钮，将口令保存到安全的地方

### 2. 文件脱敏

配置好全局口令后：
- 在文件脱敏时，系统会自动使用全局口令加密映射文件
- 不再需要每次手动设置口令
- 直接点击"执行脱敏并保存"即可

### 3. 文件还原

- 系统会自动尝试使用全局口令解密映射文件
- 如果全局口令正确，直接进入确认步骤
- 如果全局口令不匹配，会提示您手动输入正确的口令

## 注意事项

1. **口令长度**：必须至少32位字符
2. **口令保管**：请妥善保管口令，丢失后无法恢复原始数据
3. **口令同步**：口令会保存到 localStorage 和数据库，跨设备需要重新配置
4. **临时口令**：如果没有配置全局口令，脱敏时仍可以临时设置口令（仅用于当前文件）

## 技术细节

### 修改的文件

1. `web/app/components/data-masking/sandbox-config.tsx` - 添加了口令配置 UI
2. `web/app/components/data-masking/file-masking.tsx` - 自动使用全局口令
3. `web/app/components/data-masking/file-restore.tsx` - 自动尝试全局口令解密
4. `web/i18n/zh-Hans/data-masking.json` - 中文国际化
5. `web/i18n/en-US/data-masking.json` - 英文国际化

### 存储位置

- **localStorage**: `mapping_encryption_passphrase`
- **数据库**: 通过 `/console/api/user-config` API 同步

### 加密算法

- AES-256-GCM
- 随机 IV（12字节）
- 认证标签（16字节）
